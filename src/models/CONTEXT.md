# src/models/

Single source of truth for all TypeScript interfaces, error classes, and configuration constants used across the extension.

## Modules

| File | Contains |
|------|----------|
| `types.ts` | All interfaces: `SymbolInfo`, `CursorContext`, `AnalysisResult`, `TabState`, `DataKindInfo`, `DiagramEntry`, `QAEntry`, `NavigationEntry`, `NavigationHistoryState`, `PinnedInvestigation`, message types, cache/index types, LLM request types, `SYMBOL_KIND_PREFIX` map |
| `errors.ts` | `CodeExplorerError` base class + subclasses (`LLMError`, `CacheError`, `AnalysisError`, `SystemError`), `ErrorCode` enum, helper functions `isCodeExplorerError()`, `getUserMessage()` |
| `constants.ts` | Extension IDs, command names (`COMMANDS`), config keys (`CONFIG`), cache constants (`CACHE`), queue defaults (`QUEUE`), supported languages, analysis version, timeout constants |

## Key Types

- **`SymbolKindType`** — Union of `'class' | 'function' | 'method' | 'variable' | 'interface' | 'type' | 'enum' | 'property' | 'parameter' | 'struct' | 'unknown'`. Includes `'struct'` for C/C++/Rust/Go/C# structs.
- **`SymbolInfo`** — Primary input to the analysis pipeline. Contains name, kind, filePath, position, optional range, containerName, scopeChain.
- **`CursorContext`** — Lightweight input for the cursor-based flow. Contains word, filePath, position, surroundingSource (±50 lines), cursorLine. Avoids expensive VS Code symbol resolution.
- **`AnalysisResult`** — Complete output including overview, callStacks, usages, dataFlow, relationships, functionSteps, subFunctions, functionInputs, functionOutput, classMembers, memberAccess, variableLifecycle, dataKind, diagrams, qaHistory, relatedSymbols, and metadata.
- **`DiagramEntry`** — Mermaid diagram generated by the LLM. Contains title, type (flowchart/sequenceDiagram/classDiagram/stateDiagram), and mermaidSource.
- **`QAEntry`** — A single Q&A exchange from the Enhance feature. Contains question, answer, and timestamp.
- **`DataKindInfo`** — Classifies what kind of data a variable holds (label, description, examples, references). Used for variable analysis.
- **`TabState`** — UI state for a single sidebar tab (loading/ready/error/stale). Includes optional `enhancing` flag (for Q&A in progress) and `notes` field (user-added notes).
- **`LoadingStage`** — Granular progress stages: `'resolving-symbol'`, `'cache-check'`, `'reading-source'`, `'llm-analyzing'`, `'writing-cache'`.
- **`NavigationEntry`** — A single navigation step (fromTabId, toTabId, trigger, timestamp, symbolName, symbolKind). Forms the breadcrumb trail.
- **`NavigationTrigger`** — Union type: `'explore-command' | 'symbol-link' | 'sub-function' | 'caller' | 'relationship' | 'type-link' | 'breadcrumb' | 'history-back' | 'history-forward' | 'tab-click' | 'session-restore'`.
- **`NavigationHistoryState`** — Full navigation state pushed to webview (entries, currentIndex, pinnedInvestigations, currentInvestigationName/Id/Dirty).
- **`PinnedInvestigation`** — Named, saved breadcrumb trails with trail symbol metadata for restoration.
- **`ExtensionToWebviewMessage`** — `setState` (tabs + activeTabId + navigationHistory) or `showDependencyGraph` (mermaidSource + stats).
- **`WebviewToExtensionMessage`** — Includes: `ready`, `tabClicked`, `tabClosed`, `refreshRequested`, `navigateToSource`, `retryAnalysis`, `exploreSymbol`, `navigateToSymbol`, `enhanceAnalysis`, `requestDependencyGraph`, `requestSymbolGraph`, `closeDependencyGraph`, `historyBack`, `historyForward`, `pinInvestigation`, `unpinInvestigation`, `restoreInvestigation`, `reorderTabs`, `updateNotes`, `saveInvestigation`, `saveInvestigationAs`, `renameInvestigation`.

## Error Hierarchy

```
CodeExplorerError (base)
  +-- LLMError         (LLM_UNAVAILABLE, LLM_TIMEOUT, LLM_RATE_LIMITED, LLM_PARSE_ERROR, LLM_AUTH_ERROR)
  +-- CacheError       (CACHE_READ_ERROR, CACHE_WRITE_ERROR, CACHE_CORRUPT, INDEX_CORRUPT)
  +-- AnalysisError    (SYMBOL_NOT_FOUND, ANALYSIS_TIMEOUT, FILE_NOT_FOUND, LANGUAGE_NOT_SUPPORTED)
  +-- SystemError      (WORKSPACE_NOT_OPEN, DISK_FULL, PERMISSION_DENIED)
```

All LLM/cache/analysis errors are `recoverable: true`. System errors are `recoverable: false`.

## Do NOT

- Add new constants outside `constants.ts`
- Create new error types without extending `CodeExplorerError`
- Import from `webview/` (separate TypeScript project)
