## 2. Issues Identified

1. **Section ordering mismatch** (`SkillInstaller.ts`): The skill template had sections in wrong order. The actual `_serialize()` output order is: Overview → Key Points → Callers → Usage → Relationships → Data Flow → Variable Lifecycle → Data Kind → Class Members → Member Access → **Diagrams** → Steps → Sub-Functions → Function Input → Function Output → Dependencies → Usage Pattern → Potential Issues → Related Symbols → Q&A. The old skill had Steps/Sub-Functions before Class Members, and Diagrams after Function Output.

2. **Missing sections** (`SkillInstaller.ts`): The skill did not document:
   - **Usage (references table)** — `## Usage (<N> references)` with a markdown table
   - **Relationships** — `## Relationships` with type hierarchy entries
   - **Q&A History** — `## Q&A` with `json:qa_history` block (generated by Enhance feature)

3. **Insufficient explanatory content**: The skill was heavily template-based but didn't explain *why* section ordering matters, what the tagged JSON fences are, or how the extension parses the files. This made it hard for agents to understand and debug problems.

4. **Missing reference tables**: No quick-reference table for JSON block tags, no YAML frontmatter field documentation, no troubleshooting guide.

5. **Section inclusion table was misaligned**: Listed sections in procedure order (3d-3q) rather than cache file order, making it confusing to cross-reference.
